## 2. Remainder closed in this pass

### 2.1 Offline OpenAPI × auth inventory

Every path in `docs/openapi.json` is classified against the middleware:

| Class | Rule | Gate |
|-------|------|------|
| exempt | health / docs / metrics / node ingest | `_is_exempt_path` |
| admin | `/v1/admin/*`, `/admin/*` | middleware skip + `require_admin_key` |
| tenant | everything else | X-API-Key or explicit `AGENTFLOW_AUTH_DISABLED` |

**Test:** `tests/unit/test_openapi_auth_surface.py`  
**Finding:** no orphan public tenant paths; all admin OpenAPI paths sit under `/v1/admin`.  
**Gap (documented, not a code defect):** OpenAPI currently does not declare
`securitySchemes` / per-operation `security` — enforcement is middleware-based.
Schemathesis still hits live routes with a real key in CI.

### 2.2 Adversarial NL-SQL fuzz

Hypothesis fuzz over schema-qualified tables and forbidden function names in
projection and filter positions.

**Test:** `tests/unit/test_sql_guard_adversarial_fuzz.py`  
**Result:** all generated vectors raise `UnsafeSQLError` as expected.

### 2.3 Expanded ClickHouse transpile matrix

Additional shapes under tenant schema qualification:

- `COUNT(*) FILTER (WHERE …)` + `INTERVAL` (metric template)
- `LEFT JOIN` tenant-scoped tables
- multi-CTE join
- escaped quote payloads inside string literals
- classic injection payloads kept inside literals through transpile

**Test:** `tests/unit/test_clickhouse_backend.py` (expanded parametrize + injection case)  
**Result:** table references preserve `acme` qualifier; multi-statement refuse path unchanged.
